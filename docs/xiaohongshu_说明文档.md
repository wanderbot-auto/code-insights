# Code Insights 小红书说明文档

## 一句话定位
Code Insights 是一个离线、终端优先的代码库分析工具：输入本地仓库路径，快速得到语言分布、代码行统计、最大源文件与实时变更信息。

## 项目核心功能
- 基础代码统计
- 按语言统计文件数、总行数、有效代码行（剔除空行和注释）
- 项目结构与体量感知
- 最大源文件列表，快速定位“重文件”
- 缓存加速
- 基于文件哈希复用分析结果，提升重复扫描速度
- 终端可视化
- Rich 风格仪表盘、语言分布图、趋势线、活动变更区
- 可选实时监控
- 自适应 1~3s 刷新频率，展示最近变更、增删行、5 分钟趋势
- 双语输出
- 支持 `--lang zh|en`

## 主要亮点（可作为帖子卖点）
- 纯本地离线：无需上传代码，适合私有仓库和企业内网环境
- 命令简单：`code-insights analyze <repo_path>` 即可获得完整终端报告
- 信息密度高：总览、语言分布、趋势、活动都在一个界面
- 对 AI 高改动场景友好：支持 `--monitor-profile agent`，降低高频改动噪声
- 活动区可读性强：展示最近几条文件变更记录，并带变更时间

## 适用人群
- 个人开发者：快速摸清旧项目规模与热点文件
- 团队技术负责人：做代码体量巡检与改动节奏观察
- AI 协作开发者：跟踪高频改动下的文件变化轨迹


## 小红书发布版文案（可直接复制）
最近做了个终端工具 `Code Insights`，专门解决一个痛点：
拿到一个仓库后，怎么在 10 秒内看清它的“体量、语言结构、改动节奏”。

我把它做成了离线模式，输入本地路径就能出可视化分析，不依赖云端。

它能做这些事：
- 统计各语言文件数、总行数、有效代码行
- 找出最大源文件，快速定位重文件
- 监控最近变更（带时间），避免关键信息一闪而过
- 显示近 5 分钟变化趋势，判断当前改动节奏

最常用的命令：
`code-insights analyze /path/to/repo --monitor`

如果你也经常接手旧项目、做仓库巡检，或者在 AI 辅助开发里想看实时改动，我觉得这个工具会很实用。
